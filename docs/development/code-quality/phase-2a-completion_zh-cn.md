# 质量系统 Phase 2A 完成报告（摘要）

> 涵盖 ONNX 评估结果、自匹配偏置、下一步混合评分计划。原英文长文精简为中文要点。

## 背景
- 目标：在 v8.48.x 前验证本地 ONNX 的可用性，并定义改进路径。
- 数据：4,762 条记忆，使用标签生成查询做批量评估。

## 发现
1) **自匹配偏置 ~25%**：标签生成查询与内容高度重合，导致 1.0 高分。
2) **双峰分布**：大量 1.0 与 0.0，均值 0.469。
3) **质量提升有限**：质量重排对排名影响 0-3%。
4) **成本/性能**：本地 SLM 45ms（含重排），月成本 $0。

## 结论
- ONNX 适合相对排序，不适合绝对阈值。
- 需要引入使用信号/人工反馈做混合评分。

## 建议的改进（Phase 2/2B 对齐 Issue #268）
- 混合公式：ONNX + 访问频率 + 最近性 + 标签/完整度（权重待调）。
- 用户反馈入口（👍/👎），权重 2-3× AI 分数。
- 改进查询生成，减少自匹配。
- A/B 对照与分布监控（UI 已有基础）。

## 已落实
- 文档披露限制与最佳实践。
- 质量增强保持可选，默认关闭。

## 待办
- 实现混合评分与反馈写入。
- 批量重评脚本，重新计算历史分数。
- LLM-as-Judge 评审（Phase 3）。

## 风险与缓解
- 误删高价值记忆：保持人工复核、延长保留期。
- 过高质量权重导致偏差：默认权重 0.3，逐步调优。
